## Plan: Code quality/stability fixes + settings UI redesign

Two workstreams from the audit. Both ship in one release.

---

### Workstream 1: Code quality & stability (6 HIGH + 5 MEDIUM fixes)

**1.1 — Fix `confirmDialog` listener accumulation (HIGH)**
`src/main.js` — the `{ once: true }` keydown + click listeners on `el.confirmDialog` never fire when the user resolves via a button click, so they accumulate. Fix: capture the listener references and remove them inside `done()`.

**1.2 — Add a render generation counter (HIGH)**
`src/main.js` — rapid tab switching while PDF/Excalidraw is loading can leak controllers (the `_lastRenderedId` string check is insufficient when multiple `showPdf` promises are in flight). Fix: add a monotonic `_renderGen` counter, captured per `renderActive()` call. The async `.then()` callbacks compare the captured gen against the current gen — if they differ, the result is stale and torn down.

**1.3 — Catch all fire-and-forget async calls (HIGH)**
`src/main.js` — add `.catch()` / try-catch to:
- `toggleMode()` → `renderActive()` (currently no catch; a mermaid/kaTeX throw → unhandled rejection)
- `closeTab()` calls from the tab-strip click handler (middle-click, ctx-menu actions)
- `win-maximize` handler (`await appWindow.toggleMaximize()` with no try/catch)

**1.4 — Guard `file-changed` against overwriting unsaved edits (MEDIUM)**
`src/main.js` — in the `file-changed` listener's edit-mode branch, check `doc.dirty`. If the doc has unsaved changes, don't clobber the textarea silently — show a toast ("File changed on disk — your unsaved edits were kept") and let the user decide.

**1.5 — Fix `parseFloat(lineHeight)` → NaN in find-bar (MEDIUM)**
`src/views/find-bar.js` — `getComputedStyle().lineHeight` can return `"normal"` → `parseFloat` → `NaN` → `scrollTop = NaN`. Add a `|| 20` fallback.

**1.6 — Fix React root orphan on Excalidraw load failure (MEDIUM)**
`src/views/excalidraw-viewer.js` — move `createRoot` after all imports succeed. If imports throw, no root is created, so nothing leaks. Currently `createRoot` is inside the try block but before the return — if `root.render` throws, the root is orphaned. Fix: wrap the render in the existing try/catch and call `root.unmount()` in the catch.

**1.7 — Null guards for `icoMax`/`icoRestore` (MEDIUM)**
`src/main.js` — add `if (icoMax)` / `if (icoRestore)` guards before the `.classList.toggle` calls in `syncMaxIcon()`.

**1.8 — Remove dead `PALETTE` export (MEDIUM)**
`src/views/pdf-viewer.js` — the exported `PALETTE` constant is never imported anywhere (colors come from HTML `data-color` attributes). Remove the export + the `export { PALETTE }` line.

**1.9 — Add NaN guard for `parseInt(dataset.pageNum)` (LOW)**
`src/views/pdf-viewer.js` — add `|| 0` fallback on page-num parsing. Not a live bug today but defensive.

**1.10 — Make `escapeHtml` consistent (LOW)**
Create a shared `src/lib/escape.js` with a proper `escapeHtml` that escapes `& < > " '` (quotes included). Replace the inline copies in `pdf-viewer.js` and `excalidraw-viewer.js`. `tabs.js` already has a correct version — consolidate to one.

### Workstream 2: Settings UI redesign

**2.1 — Section grouping in HTML (`index.html`)**
Group the 8 settings into 3 sections with headers:
- **General** — New tab format, New tab opens in
- **Appearance** — Theme, Font size, Line spacing
- **Editor** — Line numbers, Find: match case, Close button action

Add `<div class="setting-section">` wrappers with `<div class="setting-section-title">` headers. Add `.setting-divider` between rows within a section (subtle hairline).

**2.2 — Settings CSS overhaul (`src/styles/base.css`)**
- `.setting-section` — a card-like group with `background: var(--surface)`, `border-radius: var(--radius)`, `padding`, and a subtle border. Settings within a section share a visual container (like macOS System Settings panels).
- `.setting-section-title` — 11px uppercase, `font-weight: 600`, `letter-spacing: 0.05em`, `color: var(--fg-muted)`, `margin-bottom: 8px`.
- `.setting-row` (renamed from `.setting`) — full-width row with `padding: 10px 14px`, `border-radius: var(--radius-sm)`, hover background. Rows within a section are separated by 1px dividers (`.setting-row + .setting-row { border-top: 1px solid var(--border-subtle) }`).
- Normalize control heights: toggle track from 22px → 24px to better match the 30px selects and ~28px segs. Adjust thumb transform accordingly.
- `.settings-footer` — sticky bottom with the reset button getting a subtle `color: var(--fg-muted)` (de-emphasized) and the Done button keeping primary styling. Add a top border to separate the footer from the scrollable body.
- Responsive: `@media (max-width: 520px)` — stack `.setting-row` vertically (label on top, control below, `flex-direction: column; align-items: flex-start`).

**2.3 — Update `syncSettingsControls` and reset handler**
No JS logic changes needed — the HTML restructuring (class rename `.setting` → `.setting-row`, added section wrappers) doesn't change the control IDs or the handler wiring. The `setSegActive` querySelector works unchanged because it queries by `data-setting` attribute.
