# Plan: Three new capabilities (v0.17.0) — CSV viewer, PDF export, Image annotation

Three independent features, each fills a real gap (none overlap existing functionality). Ships as MINOR. All three are additive and live in distinct parts of the codebase — no cross-dependencies.

- **CSV viewer**: ~250 LOC across 4 files, no new deps, no Rust changes
- **PDF export**: ~150 LOC + a print stylesheet, no new deps, no Rust changes (uses `window.print()` + the OS print dialog's "Save as PDF")
- **Image annotation**: ~350 LOC across 3 files + one tiny Rust command, extracts the drawing core out of pdf-viewer.js into a shared module

Combined: ~750 LOC, 1 small Rust command, **0 new npm/cargo dependencies.**

---

## Feature 1 — CSV/TSV viewer (sortable + filterable table)

Open any `.csv` or `.tsv` → renders as a clean sortable, searchable table with a sticky header. Click a column to sort (toggle asc/desc), type in a filter box to narrow rows. Handles quoted CSV fields with embedded commas, newlines, and escaped quotes (`""`).

**Why this is the right shape:** CSVs already open today — they fall into `CODE_EXT_RE` and render as plain highlighted text. The new viewer intercepts them earlier and gives them a purpose-built UI.

### Changes
- **`src/lib/documents.js`** — Add `isCsvPath()`. Add `csv` flag to `createDocument` (derive from path like the other flags, exclude from `isCode`). **Remove `csv|tsv` from `CODE_EXT_RE`** so they no longer match `isCodePath`. Persist `csv` in `serialize()`/`restore()`.
- **`src/lib/renderer.js`** — Add `renderCsv(text, { tsv })`. RFC-4180-style parser (handles quotes, embedded commas/newlines, `""` escapes). Returns sanitized HTML: a toolbar (filter input + count) + scrollable `<table>` with sticky headers.
- **`src/views/csv-viewer.js`** (new) — Controller wiring column sort (▲/▼, numeric vs lexical) and substring filter. Holds parsed rows in closure. Returns `{ destroy() }` for teardown.
- **`src/main.js`** — Add `if (doc.csv) { … return; }` branch in `renderActive()` (between Excalidraw and Code). Store controller in `_activeCsv`, tear down in outgoing-doc block. Update flag checks at lines 329, 928, 1174, 1189, 2469.
- **`src/styles/content.css`** — `.csv-viewer` styles: sticky toolbar/header, hover, sort indicators.
- **`src/lib/file-type.js`** — Add a `csv` token + small table-grid glyph.
- **`test/csv.test.js`** (new) — Quoted fields, escaped quotes, embedded newlines, TSV mode, empty input, HTML sanitization.

## Feature 2 — Export Markdown to PDF

One click → OS print dialog with a print-only stylesheet applied. User picks "Save as PDF". Reuses everything `exportHtml` already builds.

**Why `window.print()` and not a Rust PDF crate:** zero new deps, zero binary-size growth, identical on Windows/macOS/Linux via the native dialog. The existing `exportHtml` already assembles a styled standalone HTML doc we can render off-screen for printing. Trade-off: user picks "Save as PDF" in the dialog rather than silent file write. Right v0.17.0 trade-off.

### Changes
- **`src/styles/content.css`** — New `@media print { ... }` block: hides all chrome (`.app-header, #sidebar, #toc, .tab-strip, .tool-group, .pdf-draw-toolbar, .reading-progress, .find-overlay, .copy-btn, .anchor-link, .fold-toggle, .editor-status`), resets `#document` for natural flow, `@page { margin: 18mm }`, `pre { white-space: pre-wrap }`, `break-inside: avoid` for code/headings/rows, `print-color-adjust: exact` so hljs colors survive.
- **`src/main.js`** — Add `async function exportPdf()`: same guards as `exportHtml`; sync editor content; call `await showDocument(el.document, doc.content)` to ensure Mermaid + extra languages are rendered; `window.print()`; restore prior mode. Add `el.exportPdf` + event listener + show/hide in renderActive (mirrors `el.export`). Add command-palette entry.
- **`index.html`** — Add a second icon-only button `#btn-export-pdf` next to `#btn-export`. (Simpler than a dropdown menu, scales later.)
- **No Rust changes** — `window.print()` is webview-only; OS dialog handles output.

## Feature 3 — Image annotation (draw on images)

Open any image → click "Annotate" → floating toolbar (pen/highlighter/eraser + color palette + clear + save). Draw on a transparent overlay canvas. Save writes a new PNG with annotations baked in.

**Why this works well:** the PDF drawing code in `pdf-viewer.js` is already self-contained. We extract it into a shared module both PDFs and images use.

### Changes
- **`src/lib/drawing.js`** (new) — Extracted drawing core: `createStrokeController()`, `attachPointer()`, `renderStrokes()`, `drawSingleStroke()` (quadratic-curve smoother), `pointInStroke*()` (eraser). Strokes in normalized 0..1 coords. Constants: `PEN_WIDTH=2`, `HIGHLIGHT_WIDTH=14`, 5-color palette.
- **`src/views/pdf-viewer.js`** — Refactored to import from `src/lib/drawing.js`. No behavior change for PDFs (pure extraction).
- **`src/views/image-viewer.js`** (new) — Export `showImage(container, filePath)` returning `{ setDrawMode, setTool, setColor, clearAll, saveAnnotations, destroy }`. Builds `<img>` + absolutely-positioned overlay canvas. `saveAnnotations()` composites image+strokes at full original resolution via `canvas.toBlob('image/png')`, passes bytes to new Rust command.
- **`src/main.js`** — Replace inline image-rendering code (lines 368-396) with `showImage(...)`; store controller in `_activeImage`. Generalize `pdfSelectTool/pdfSelectColor/pdfToggleToolbar` to dispatch on `_activePdf || _activeImage`. Un-hide `el.draw` in the image branch; update its title to "Annotate".
- **`src-tauri/src/commands.rs`** + **`lib.rs`** — Add `save_annotated_image(bytes: Vec<u8>, suggested_name: String) -> Result<String, String>`: rfd dialog filtered to `.png`, writes bytes, returns path. Register next to `save_image`.
- **`src/styles/content.css`** — `.image-annot-overlay` styles (mirror `.pdf-draw`). Save button reuses `.pdf-tool-btn`.
- **`test/drawing.test.js`** (new) — Stroke model round-trips, hit-testing for pen vs highlighter thresholds, undo/clear, no-throw on edge cases (empty, single-dot, 2-point, 3+ point strokes).

## Release as v0.17.0 (MINOR)

## Out of scope
- CSV editing (view-only)
- CSV formulas/pivot tables (not Excel)
- Silent no-dialog PDF generation (defer until asked)
- Per-image annotation persistence (sidecar JSON) — annotations flatten into saved PNG; original untouched
- SVG export of annotations — PNG only

## Implementation order
1. **CSV viewer** (smallest, no Rust, no shared code)
2. **PDF export** (no Rust, no shared code)
3. **Image annotation** (largest; depends on extracting drawing core, which touches pdf-viewer.js — do last so regressions are isolated)
