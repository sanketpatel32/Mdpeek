## Plan: Embed full Excalidraw as a new tab type

**Goal:** A new "Excalidraw" tab type that gives users the full Excalidraw canvas ‚Äî shapes, text, arrows, freehand, selection, export. Opens from the `+` button (settings ‚Üí new tab format ‚Üí Excalidraw) or by opening a `.excalidraw` file. Lazy-loaded so markdown/PDF users pay zero startup cost.

### Weight reality check (honest)

| Component | Size | When loaded |
|---|---|---|
| `@excalidraw/excalidraw` | **344 KB gzip** (1.1 MB min) | Only when an Excalidraw tab is opened |
| `react` + `react-dom` | **~45 KB gzip** combined | Only when an Excalidraw tab is opened |
| **Installer** | 4.48 MB ‚Üí **~5.3 MB** (+~800 KB bundled into `dist/`) | Always shipped, fetched on demand |
| Startup / markdown path | **Unchanged** | Excalidraw code is lazy `import()` ‚Äî never touches initial load |

5.3 MB is still dramatically smaller than Typora (~80MB), Obsidian (~80MB), or VS Code (~90MB). The tradeoff: Excalidraw is a heavy library, and there's no way around that if you want the full experience.

### How it mounts

Excalidraw is a React component, but mdpeek doesn't use React. The integration uses React imperatively (no JSX, no build pipeline change):

```js
// src/views/excalidraw-viewer.js (lazy-loaded module)
const React = (await import('react')).default;
const ReactDOM = (await import('react-dom/client')).default;
const { Excalidraw } = await import('@excalidraw/excalidraw');

const root = ReactDOM.createRoot(container);
root.render(
  React.createElement(Excalidraw, {
    initialData: savedScene,
    onChange: (elements, appState) => { /* save debounce */ },
  })
);
```

This mounts Excalidraw into a `<div>` inside `#document` (the same container used for markdown view mode). On tab switch, `root.unmount()` tears it down cleanly.

### Files touched (7) + 3 new dependencies

**New dependencies** (all lazy-loaded, zero impact on initial bundle):
- `react`, `react-dom` (peer deps for Excalidraw)

- `export async function showExcalidraw(container, initialData)` ‚Äî lazy-imports React + ReactDOM + Excalidraw, creates a `ReactDOM.createRoot`, renders `<Excalidraw>` into the container. Returns a controller `{ getSceneData(), exportPng(), exportSvg(), destroy() }`.

### The Excalidraw UX in mdpeek
- **New tab** ‚Äî Settings ‚Üí New tab format ‚Üí Excalidraw, then click `+`. Or Ctrl+N if that's the saved format.
- **The canvas** fills the document pane. Full Excalidraw toolbar (shapes, text, arrows, freehand, images, eraser, laser, pan, selection, export).
- **Auto-save** ‚Äî the scene is saved to `doc.content` as JSON (debounced 1s). It persists across tab switches within the session. If you close the tab without saving to disk, you'll get the unsaved-changes warning.
- **Save to disk** ‚Äî Ctrl+S saves the `.excalidraw` JSON file. The file is a standard Excalidraw scene, openable in any Excalidraw instance (web, VS Code plugin, etc.).
- **Export** ‚Äî Excalidraw's built-in export menu (hamburger menu ‚Üí Export) works for PNG/SVG. This is Excalidraw's own UI, no custom code needed.
- **No edit/view toggle** ‚Äî Excalidraw is always interactive (it IS the editor). The mode toggle is hidden.
