## Feature 1 — Folder-wide search (grep across a folder)

Right-click a folder in the file explorer → "Search in folder" → a search panel opens with an input + scrollable results grouped by file. Each result shows the matching line with the match highlighted. Click a result → opens the file and jumps to the matching line.

**Why Rust-side grep:** avoids O(files) IPC round-trips, skips binary files cheaply, and reuses the same skip-dir/dotfile rules as `list_dir` so the search matches what the tree shows. `regex` crate is NOT added — v0.18.0 ships plain substring search (case-sensitive toggle). Regex can come later.

### 1a — Rust command (`src-tauri/src/commands.rs`)
Add `search_in_folder(root, query, case_sensitive, max_results) -> SearchSummary`:
- Structs: `SearchMatch { line, column, text, match_start, match_end }`, `FileSearchResult { path, matches }`, `SearchSummary { results, truncated, total_matches, files_scanned, files_with_matches }`.
- **Recursive walk** via `std::fs::read_dir` (fresh code — `list_dir` is single-level).
- **Reuse skip rules** from `list_dir` (dotfiles + `SKIP_DIRS = node_modules, target, dist, build, __pycache__`). Extract into a shared `fn is_skipped_entry(name, is_dir)`.
- **Binary rejection, two layers**: (1) extension blocklist (extend existing list with archives, media, fonts, binaries), (2) NUL-byte heuristic on first 8 KB.
- **Decode** with `String::from_utf8_lossy`; scan line-by-line with case-folded `contains`.
- **Limits**: default `max_results = 1000`, `files_with_matches ≤ 200`. Set `truncated = true` when hit.
- **Empty query**: return empty `SearchSummary` (UI calls on every keystroke with debounce).
- Register in `src-tauri/src/lib.rs` `invoke_handler!`.

### 1b — Search panel module (`src/views/folder-search.js`, new)
Self-contained panel controller `initFolderSearch(container, onOpen)` returning `{ open(folderPath), close(), destroy() }`:
- Builds DOM once (idempotent like find-bar's `initFindBar`).
- **Layout**: header row (search input + case-toggle + count + close), scrollable `.folder-search-results` list.
- **Search loop**: debounced (200ms) `invoke('search_in_folder', ...)`. Generation counter so stale slow queries don't overwrite newer ones.
- **Render results**: grouped by file as a sticky sub-header (path + match count) followed by match rows. Each match: `<div class="search-match"><span class="search-line">42</span><span class="search-text">… <mark>match</mark> …</span></div>`. Click → calls `onOpen(path, line)`.

### 1c — Sidebar integration (`index.html` + `src/main.js`)
- **Markup**: `<aside id="folder-search" class="folder-search-panel hidden">` inside `#view-mode` as a sibling of `#toc`.
- **Tree right-click hook**: wire directly in main.js on `el.fileTreeBody` (the existing second-listener pattern). On `contextmenu` of `.tree-row[data-kind="dir"]`, call `folderSearch.open(path)`. Skip a fancy context menu — direct action is simpler.
- **Toolbar button**: `#btn-search` next to `#btn-explorer` opens the panel — searches the current explorer root, or prompts to pick a folder if none.
- Wire `onOpen(path, line)` to `openPathAndJump(path, line)`.

### 1d — Line-jump extension to `openPath` (`src/main.js`)
Extend `openPath(path, content, opts = {})`:
- `opts.line` → set `doc.pendingLine = line` after `store.open`.
- Post-render hook consumes `pendingLine` based on the doc's mode.

### 1e — Post-render line-jump hook (`src/main.js`)
After each `renderActive()` branch's render + scroll-restore, check `doc.pendingLine`:
- **Edit mode**: char offset from line + find-bar's existing line→scrollTop math → `editor.setState({ start, end, scrollTop })`.
- **Code view mode**: gutter div's line-height × (line - 1) → `el.document.scrollTop`. No DOM change.
- **Markdown view mode**: fall back to opening at top + surfacing the query via the find bar (`find.open(query)`) so in-doc search takes over.
- Clear `doc.pendingLine` after handling.

## Feature 2 — Pinnable tabs

Each tab gets a pin/unpin via right-click context menu. Pinned tabs:
- Shrink to icon-only (~30px wide), title and close button hidden.
- Stick to the left of the tab strip.
- Persist across sessions (serialize writes them first).
- Survive "close all" / "close others" / "close right" bulk actions.

### 2a — Document model (`src/lib/documents.js`)
- Add `pinned = false` to `createDocument` destructured params.
- Add `pinned` to returned object.
- Persist in `serialize()`: add `pinned: d.pinned || false` + sort docs so pinned come first (stable).
- Restore in `restore()`: add `pinned` field. Sort-from-serialize means restored order already has pinned first.

### 2b — Tab strip render (`src/views/tabs.js`)
Update template (lines 27-38):
- Add `pinned` class when `d.pinned`.
- Hide title + close button via CSS for pinned tabs.
- Keep icon. Dirty indicator becomes an overlay dot.

### 2c — Reorder on pin/unpin (`src/lib/documents.js`)
Add `setPinned(id, pinned)` method:
1. Update `doc.pinned`.
2. Stable-sort `this.docs` so pinned come first.
3. Emit `change`.

### 2d — Context menu integration (`index.html` + `src/main.js`)
- **index.html `#ctx-menu`**: add `<button class="ctx-item" data-action="pin">Pin tab</button>` at the top.
- **contextmenu handler** (main.js:2079-2107): relabel pin item to "Unpin tab" when target tab is already pinned.
- **`ctxAction` dispatcher** (main.js:790-803): add `else if (action === 'pin')` → `store.setPinned(tabId, !doc.pinned)`.

### 2e — Bulk-close skips pinned (`src/main.js`)
Update `ctxAction`:
- `close-all`: `store.docs.filter((d) => !d.pinned)`.
- `close-others`: `id !== tabId && !d.pinned`.
- `close-right`: also skip pinned.
- `closeDocs` itself unchanged — filtering happens at call sites.
- Edge: if all tabs pinned → close-all is a no-op. Existing "always keep one tab" logic still applies.

### 2f — Session restore preserves pinned untitled tabs (`src/main.js`)
Filter at line 2668: add `&& !s.pinned` so a pinned untitled tab survives restart.

### 2g — CSS (`src/styles/base.css`, near tab rules ~line 1382)
- `.tab.pinned` — `max-width: 40px; min-width: 32px; padding: 0 6px;` (icon-centered).
- `.tab.pinned .tab-title` — `display: none`.
- `.tab.pinned .tab-close` — `display: none`.
- `.tab.pinned .tab-icon` — `margin: 0;`.
- `.tab.pinned .tab-dot` — `position: absolute; top: 3px; right: 4px; font-size: 7px;`.
- `.tab.pinned.active::after` — adjust underline insets for narrower width.
- Divider: `.tab.pinned + .tab:not(.pinned)` → `border-left: 1px solid var(--border-subtle); margin-left: 4px;`.

### 2h — Tests (`test/documents.test.js`)
- `setPinned(id, true)` flips the flag + moves doc before all unpinned (stable within group).
- Pinning an already-pinned tab is a no-op.
- `serialize()` writes pinned docs first.
- `restore()` reads `pinned` back correctly.
