## Plan: Settings modal with all preferences

**Goal:** A gear-icon button in the toolbar opens a centered modal dialog listing every user-tunable preference in one place. The headline setting (per your request): **default new-tab format** — Markdown vs Plain Text, so clicking `+` creates the kind of doc you want.

### Settings surfaced (6 total)

| Setting | Key | Control | Values |
|---|---|---|---|
| **New tab format** *(new)* | `mdpeek-new-tab-format` | two segmented buttons | `markdown` (default) \| `text` |
| **New tab opens in** *(new)* | `mdpeek-new-tab-mode` | segmented buttons | `view` (default) \| `edit` |
| **Theme** | `mdpeek-theme` | dropdown (reuse theme list) | light/dark/solar-light/solar-dark/dracula/nord |
| **Close button action** | `mdpeek-close-action` | radio | `ask` (default) \| `tray` \| `quit` |
| **Find: match case** | `mdpeek-find-case` | toggle | `0` (default) \| `1` |
| **Reset to defaults** | — | danger button | clears all the above |

Zoom and sidebar stay as in-app toolbar controls (they're inherently visual/positional and don't make sense in a settings sheet). Session (`mdpeek-session`) is internal, not shown.

### Interaction details
- **Live apply:** theme, find-case, and close-action take effect immediately when changed in settings (no need to close-and-reopen). New-tab format/mode apply to the next new tab (can't retroactively change existing tabs — standard behavior).
- **Idempotent wiring:** the gear button and all modal controls are wired once at module load (same pattern as the close/confirm dialogs). Opening settings just reads current values + flips `hidden`.
- **Esc behavior:** Esc inside settings closes the modal (with stopPropagation so it doesn't cascade to the find bar's Esc handler or the textarea).
- **Reset to defaults:** clears all 5 keys, calls `applyTheme('light')`, sets find-case off, removes close-action key, updates all modal controls to default states. Does NOT touch zoom, sidebar, or session (those aren't in the settings sheet).

### What stays unchanged
- Existing toolbar buttons, window controls, theme dropdown (the settings modal's theme select is an additional access point to the same `applyTheme`; both stay in sync because they read/write the same key).
