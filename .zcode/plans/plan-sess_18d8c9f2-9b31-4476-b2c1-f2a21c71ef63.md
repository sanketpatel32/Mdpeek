# Plan: Settings Modal — Visual Redesign (Calm Glass)

## Goal
Re-skin the Settings dialog to match the v0.32.0 "calm glass" aesthetic the rest of the chrome got. **Visual only — no behavior changes, no new settings, no Reset-scope changes.** The modal currently uses flat opaque surfaces and cramped rhythm while the rest of the app (topbar, sidebar) is frosted and spacious.

## Source-of-truth map (confirmed via exploration)
- HTML: `#settings-dialog` in `index.html:448–916` — overlay → `.modal.modal-wide` → `h2.modal-title` + `.settings-shell` (grid: `.settings-sidebar` nav + `.settings-content` panels) + `.modal-actions.settings-footer`.
- CSS: all in `base.css` — `.modal-overlay` (2443), `.modal`/`.modal-wide` (2455/2867), `.settings-shell` (3339), `.settings-sidebar`/`.settings-cat` (3347/3357), `.settings-content` (3395), `.setting-card`/`.setting-row`/`.setting-label`/`.setting-desc` (3403–3438), `.settings-footer` (3440), `.setting-select` (3521), `.seg`/`.seg-btn` (3486), `.setting-toggle`/`.toggle-track`/`.toggle-thumb` (3875). Motion overrides in `motion.css`.
- Tokens available: `--bg-elevated`, `--surface`, `--surface-hover`, `--surface-active`, `--border`, `--border-subtle`, `--accent`, `--accent-soft`, `--shadow-sm/md/lg`, `--radius-sm/-/lg`, `--ease-spring/out/in`, `--dur-1..4`. All per-theme.

## Scope (visual-only changes)

### 1. Frosted glass dialog card (the centerpiece)
`.modal-wide` (the settings card) gets the same treatment as the topbar:
```css
.modal-wide {
  background: color-mix(in srgb, var(--bg-elevated) 86%, transparent);
  backdrop-filter: blur(40px) saturate(180%);
  -webkit-backdrop-filter: blur(40px) saturate(180%);
  box-shadow: 0 24px 64px rgba(0,0,0,0.18), 0 8px 24px rgba(0,0,0,0.10), var(--shadow-lg);
}
@supports not ((backdrop-filter: blur(1px)) or (-webkit-backdrop-filter: blur(1px))) {
  .modal-wide { background: var(--bg-elevated); }
}
```
Heavier blur than the topbar (40px vs 20px) since the dialog floats above content and wants to feel like a distinct elevated layer. Opaque fallback for safety. Only `.modal-wide` — leave plain `.modal` (the small confirm dialogs) opaque so this change is scoped to Settings.

### 2. Header treatment
- `.modal-title` ("Settings") → bigger (19px), tighter margin, with a subtle bottom hairline so it reads as a header bar separating title from the two-pane shell. Add small muted subtitle is out of scope (no text change), just spacing + divider.

### 3. Sidebar (category nav) — macOS System Settings feel
- Wider (168px → 180px), more breathing room, rounded "pill" active state.
- Active category: filled pill with `--accent-soft` + accent text (already there) + a 3px accent bar on the left edge (the signature macOS sidebar accent).
- Hover: softer `--surface-hover` with the existing transition.
- Add `:focus-visible` ring (currently missing entirely) — `outline: 2px solid var(--accent); outline-offset: -2px;`.
- Icons: render at consistent 16px with `opacity: 0.9`, full opacity on active.

### 4. Content pane polish
- `.settings-content`: drop the harsh `border-left` divider (the frosted card + spacing reads cleaner without it); replace with a subtle `box-shadow` inset hairline on the sidebar edge OR just `gap` in the grid. Increase left padding.
- Add a **per-panel title** that mirrors the active category (e.g. "Appearance", "Editor") — a small muted uppercase label at the top of each panel. This is a visual addition via CSS `::before` driven by `data-cat`, so **no HTML/JS change needed** — purely cosmetic. (If `::before` content-per-attr proves fragile, I'll add a static `<h3>` per panel instead. Either way, visual only.)
- `.setting-card`: soften — remove the heavy `box-shadow` (redundant inside the frosted card), keep the hairline border + radius. Background stays `--bg-elevated` so cards read as inset wells against the translucent shell.

### 5. Row rhythm + typography
- `.setting-row` padding `11px 14px` → `14px 16px` (less cramped). Gap `16px` → `18px`.
- `.setting-desc` font-size `11.5px` → `12px` (back above the 12px floor; 11.5 was a half-pixel oddity). Line-height `1.4` → `1.45`.
- `.setting-label` stays 13px/600.
- `.setting-row:hover` keeps `--surface-hover` but slower (`var(--dur-1)`).

### 6. Form controls refinement
- `.setting-select`: `:focus` → `:focus-visible` (so the border-color change doesn't fire on mouse click). Add a subtle `box-shadow: 0 0 0 3px var(--accent-soft)` ring on focus-visible for the calm-glass look. Fix the hardcoded chevron color (`%2386868b`) → use `currentColor` so it themes.
- `.seg` / `.seg-btn`: add `:focus-visible` ring to `.seg-btn` (missing). Active seg already has `box-shadow: var(--shadow-sm)` — keep.
- `.toggle-thumb`: keep pure-white (intentional iOS look) but swap the hardcoded `rgba(0,0,0,0.25)` shadow for a token-driven `var(--shadow-sm)`.

### 7. Footer
- `.settings-footer`: actually make it visually distinct as a footer region — slightly recessed background (`var(--surface)` at low opacity via color-mix), keep the hairline top border. Buttons get the unified motion.css tactile treatment (already do via the `.modal-btn` rule). Reset button hover already goes `--danger` — keep.

### 8. Animated panel transitions
When switching categories, fade+slide the new panel in (currently instant `hidden` swap). Pure CSS:
```css
.settings-panel:not(.hidden) {
  animation: settings-panel-in var(--dur-2) var(--ease-out);
}
@keyframes settings-panel-in {
  from { opacity: 0; transform: translateY(6px); }
  to   { opacity: 1; transform: translateY(0); }
}
```
Respects reduced-motion (motion.css clamps it). No JS change — the `hidden` toggle already drives visibility, the animation just plays when `.hidden` is removed.

## Risk
Low. All changes are CSS, scoped to settings selectors, with a `@supports` fallback for backdrop-filter. The frosted card over a busy editor could reduce text contrast — mitigated by 86% opacity (high enough that `--fg` text stays fully legible on all 10 themes). The per-panel `::before` title is the only "clever" bit; if `attr(data-cat)` for `content:` doesn't theme well I'll fall back to a no-content approach and the visual still ships.
